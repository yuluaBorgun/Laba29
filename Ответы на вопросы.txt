1. Что такое Qt?

Qt (кьют) — фреймворк для разработки кроссплатформенного программного обеспечения на языке
программирования C++. Для многих языков программирования существуют библиотеки,
позволяющие использовать преимущества Qt: Python — PyQt, PySide; Ruby — QtRuby; Java — Qt
Jambi; PHP — PHP-Qt и другие

2. В чем отличие QtWidgets от QtGui?

- QtWidgets: Этот модуль предоставляет классы для создания пользовательского интерфейса и виджетов, таких как кнопки, 
текстовые поля, диалоговые окна и т. д. Это основной модуль для создания GUI-приложений в Qt.

- QtGui: Этот модуль содержит классы для работы с графикой, такими как 2D-рисование, шрифты, 
изображения и цвет. Он предоставляет функциональность для управления графическими элементами, 
но не включает в себя виджеты для интерфейса. Начиная с Qt 5, многие функции из QtGui были объединены с QtWidgets.

3. Зачем нужен модуль QtCore? Можно ли создать приложение только с помощью этого модуля?

Модуль QtCore предоставляет базовые не-GUI классы, такие как обработка событий, управление потоками, работа с файлами, время и таймеры. 
Он является основой фреймворка Qt. Несмотря на то, что многие вещи можно реализовать только с использованием этого модуля,
 полноценное приложение с интерфейсом пользователя без использования QtWidgets (или его комбинаций) будет невозможно, 
так как в QtCore нет классов для создания графических виджетов.

4. Перечислите все родительские классы для QPushButton

Класс QPushButton наследуется от следующих классов (по иерархии):
- QObject (корневой класс для всех объектов Qt)
- QAbstractButton (абстрактный класс для всех видов кнопок)
- QWidget (базовый класс для всех виджетов GUI)
- QFrame (стандартный класс для рамок, который включает в себя виджеты)
- QAbstractItemView (если рассматриваются более сложные элементы, например, для кнопок в представлениях)

▎ 5. Почему в документации к классу QPushButton нет описания сигнала clicked? Где его найти?

Документация к классу QPushButton описывает все специфические для класса сигналы и слоты, 
но сигнал clicked унаследован от его родительского класса — QAbstractButton. 
Поэтому описание сигнала clicked можно найти в документации именно для класса QAbstractButton.
В документации Qt сигналы и слоты наследуются, и общий сигнал может быть описан в базовом классе, 
чтобы избежать дублирования информации.
